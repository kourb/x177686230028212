ТЗ: «Визовый Ассистент»
Версия: 4.0  Дата: 26.03.2026  Платформа: iOS + Веб (пользовательский лендинг + веб-приложение + админ-панель)  Рынок: Россия


________________


1. Общее описание
Мобильное приложение для самостоятельной подготовки и подачи документов на визу гражданами РФ + публичный лендинг + полноценное веб-приложение для пользователей + административная веб-панель для сотрудников. Начиная с версии 4.0 поддерживаются несколько типов виз (шенгенская и другие), с возможностью расширения.


Пользователь проходит весь путь от сканирования паспорта до получения готового пакета документов и (для шенгенской визы) талона на запись в визовый центр. Дополнительно — оформление медицинской страховки как в процессе подачи на визу, так и отдельно.


________________


2. Этапы разработки
Этап 1 — iOS-приложение + Административная панель
Полный функционал мобильного приложения (iOS) + административная панель для сотрудников.
Содержание —  (разделы 4–10 данного ТЗ).
Этап 2 — Публичный лендинг + Веб-приложение для пользователей
В рамках этапа также реализуются все необходимые бэкенд-эндпоинты для поддержки веб-клиента: email-авторизация, WebAuthn, Web Push, скачивание файлов через браузер и прочие расширения существующего API.

Мои заявки:

Профиль:

Адаптации под мобайл (<768px):


* Нижний таб-бар (аналогично iOS)
* Все шаги флоу в одну колонку
* Таблицы: card view

4. Типы виз 
Начиная с версии 4.0 приложение поддерживает несколько типов виз, не ограничиваясь шенгенской. Администратор может добавлять новые типы виз через административную панель.
4.1 Поддерживаемые типы виз
На старте поддерживаются:


* Шенгенская виза (существующий функционал)
* Визы других стран (США, Великобритания, ОАЭ, Китай и др.) — добавляются через админку
4.2 Выбор типа визы пользователем
На начальном экране оформления визы пользователь выбирает:


* Тип направления: Шенген / Другая страна
* Конкретную страну назначения
* Тип визы для выбранной страны (список определяется настройками в админке)


Для шенгена доступны стандартные типы (туристическая C, деловая C, учебная C, транзитная B, национальная D).
Для остальных стран — типы виз настраиваются администратором индивидуально.
4.3 Кастомные поля анкет
Каждый тип визы имеет свой набор полей анкеты. Поля полностью настраиваются в административной панели без релиза приложения:


* Администратор создаёт набор полей для каждого типа визы
* Поддерживаемые типы полей: текст, дата, список (dropdown), загрузка файла, чекбокс, число
* Для каждого поля задаётся: название, тип, обязательность, подсказка
* Поля для шенгена остаются стандартными (см. раздел 7, Шаг 2)
* Поля для новых типов виз полностью кастомные — определяются администратором
4.4 Слоты визовых центров
Функциональность слотов доступна только для шенгенской визы. Для других типов виз шаг выбора слота отсутствует — пользователю предоставляется только готовый пакет документов.

5. Система анкет VFS Global
5.1 Управление в административной панели
В настройках административной панели добавляется переключатель (toggle) «Анкета VFS Global»:


* При включении: пользователям становится доступна возможность заполнения официальной анкеты VFS Global
* При выключении: шаг анкеты VFS скрыт
5.2 Заполнение анкеты пользователем
Когда переключатель включён, после стандартного шага заполнения анкеты появляется дополнительный шаг — заполнение анкеты VFS Global:


* Форма отображается по официальной структуре анкеты VFS Global
* Данные, уже введённые в основной анкете, автоматически подставляются в соответствующие поля
* Пользователь заполняет оставшиеся поля вручную
* Валидация полей в реальном времени
* Автосохранение черновика
________________
5.3 Передача на печать
После заполнения анкеты VFS Global пользователю доступны:


* Просмотр заполненной анкеты в приложении перед отправкой на печать
* Кнопка «Отправить на печать» — генерируется PDF анкеты VFS, открывается системный диалог печати
* Кнопка «Скачать PDF» — анкета сохраняется в виде PDF-файла (на устройство / в браузере)
* Кнопка «Отправить на email» — PDF анкеты отправляется на email пользователя

Шаг 2б — Медицинская страховка (встроенная в флоу визы)
Вариант А — загрузить свой полис:


* Загрузка файла (PDF / фото)

Вариант Б — оформить страховку прямо здесь:


* Переход на экран/страницу страховки (см. раздел 8)
* После оформления — автоматически прикрепляется к анкете

Шаг 3 — Загрузка фото
Требования к фото:


* Размер: 35×45 мм, белый фон
* Формат: JPEG, мин. 600 dpi
* Лицо занимает 70–80% кадра


Функционал:


* iOS: съёмка прямо в приложении (рамка-подсказка)
* Веб: загрузка из файловой системы или захват через веб-камеру (MediaDevices API) с рамкой-подсказкой
* Автоматическая проверка соответствия требованиям (ML-модель): фон, размер лица, тени, очки, головной убор
* При несоответствии — подсказка что исправить + переснять
* Автообрезка и подгонка под нужный размер


Важно: фото создаётся заново для каждой визы, не переиспользуется из предыдущих заявок


Для группы: фото загружается отдельно для каждого заявителя

Шаг 7 — Генерация PDF и выбор визового центра
Доступен только после апрува менеджером.


7.1 Генерация PDF анкеты:


* Бэкенд автоматически формирует PDF по официальному шаблону страны назначения
* Данные из анкеты подставляются в соответствующие поля шаблона
* PDF доступен для скачивания и отправки на email
* Кнопка «Отправить на email» — PDF анкеты + инструкция по посещению центра


7.2 Выбор визового центра и даты (только для шенгенской визы):


* Список городов с визовыми центрами: Москва, Санкт-Петербург, Екатеринбург, Новосибирск, Казань, Краснодар и др.
* Интерактивный календарь доступных слотов
* Цвет слота: зелёный — есть места, жёлтый — мало мест, серый — занято
* Выбор времени из доступных интервалов
* Для группы: бронирование слота на всех заявителей одновременно
* При отсутствии слотов: push-уведомление о появлении мест

10. Административная панель
Веб-приложение для сотрудников. Доступ — только по логину/паролю с 2FA, без публичной регистрации.
10.1 Роли
Роль
	Возможности
	Менеджер
	Просмотр и проверка заявок, апрув / запрос правок, коммуникация с пользователями, присвоение слотов
	Старший менеджер
	Всё из менеджера + управление слотами, ручное изменение статусов
	Администратор
	Полный доступ: роли, настройки, справочники, финансы, возвраты, управление типами виз и полями анкет
	10.2 Модуль: Заявки
* Таблица всех заявок с фильтрами: по статусу, стране, дате создания, менеджеру, городу подачи
* Поиск по ФИО заявителя, номеру паспорта, номеру заявки
* Сортировка по дате создания, дате записи в центр, статусу
* Подсветка заявок, требующих внимания: новые, просроченные, с автофлагами


Карточка заявки:


* Все данные анкеты и заявителей
* Загруженные документы — просмотр inline (фото паспорта, бронь, билеты, страховка)
* Фото заявителя с пометкой результата автопроверки
* История изменений статуса: кто, когда, что изменил
* Данные платежа


Проверка и решение:


* Кнопка «Апрувнуть» — заявка переходит в статус «Апрувнута», пользователю отправляется уведомление, запускается генерация PDF
* Кнопка «Запросить правки» — менеджер вводит комментарий (свободный текст или шаблон), пользователь получает уведомление с пояснением
* Шаблоны причин правок: «Паспорт нечитаем», «Фото не соответствует требованиям», «Срок паспорта истекает», «Даты не совпадают», «Отсутствует документ» и др.


Автоматические флаги:


* Срок паспорта < 6 мес. до выезда
* Даты поездки в прошлом
* Страховка не покрывает все дни поездки
* Загруженный документ нечитаем
* Фото не прошло ML-проверку
* Дублирующая заявка
* Подозрительная активность: много заявок с одного аккаунта за короткое время
10.3 Модуль: Визовые центры и слоты
Функционал слотов действует только для шенгенской визы.


Список слотов VFS Global:


* В модуле отображается таблица доступных слотов, полученных через API VFS Global
* Таблица обновляется автоматически по расписанию или вручную (кнопка «Обновить»)
* Поля: дата, время, визовый центр, город, количество доступных мест, статус


Ручное присвоение слотов:


* Менеджер открывает заявку пользователя и назначает конкретный слот из таблицы VFS Global
* После назначения пользователь получает уведомление (Push + email) с датой, временем и адресом визового центра
* Слот отмечается как занятый в таблице


Автоматическое присвоение слотов:


* Функция «Автоприсвоение» — запускается вручную менеджером или по расписанию
* Система сопоставляет заявки в очереди (статус «Апрувнута», без назначенного слота) с доступными слотами через API VFS Global
* Присвоение в порядке очереди: приоритет по дате апрува заявки (FIFO)
* После успешного присвоения — пользователю отправляется уведомление (Push + email)
* При отсутствии подходящих слотов — заявка остаётся в очереди, менеджер получает уведомление


Управление слотами вручную (существующий функционал):


* Добавление доступных дат и времени вручную (независимо от VFS API)
* Закрытие слотов
* Статистика загрузки: занятые / свободные слоты по каждому центру
10.4 Модуль: Пользователи
* Список пользователей с поиском по телефону, email, ФИО
* Карточка пользователя: контакты, дата регистрации, список заявок, история платежей
* Блокировка аккаунта с указанием причины
* Ручная смена статуса заявки в исключительных случаях — только старший менеджер / администратор
10.5 Модуль: Платежи
* Список всех транзакций с фильтрами: по статусу, методу оплаты, дате, сумме
* Поиск по номеру транзакции, ФИО, номеру заявки
* Статусы: «Успешно» / «В обработке» / «Ошибка» / «Возврат»
* Инициирование возврата через ЮMoney
* Экспорт в CSV за выбранный период
10.6 Модуль: Страховки
* Список оформленных полисов с фильтрами по дате, страховщику, статусу
* Привязка полиса к заявке на визу
* Статусы: «Активна» / «Истекла» / «Отменена»
10.7 Модуль: Настройки (только администратор)
Управление типами виз:


* Создание нового типа визы: название, страна назначения, список типов (dropdown-значения)
* Редактирование и деактивация существующих типов виз


Управление полями анкет:


* Для каждого типа визы — конструктор набора полей анкеты
* Добавление поля: название, тип (текст / дата / список / файл / чекбокс / число), обязательность, подсказка
* Изменение порядка полей (drag & drop)
* Деактивация поля без удаления (сохраняет данные существующих заявок)
* Шенгенские поля редактированию через конструктор не подлежат — зафиксированы стандартом


Управление анкетой VFS Global:


* Переключатель «Анкета VFS Global» — включить / выключить отображение шага VFS-анкеты для пользователей
* Управление шаблоном PDF анкеты VFS: загрузка актуального шаблона без релиза приложения


Существующий функционал:


* Управление сотрудниками: добавить / удалить / изменить роль
* Справочник шаблонов комментариев для правок
* Управление шаблонами PDF по каждой стране: загрузка нового шаблона без релиза приложения
* Управление списком стран Шенгена: консульский сбор, тип анкеты, обязательные документы
* Настройки уведомлений: какие события и кому отправляют письма / пуши
* Настройки валидации: пороговые значения

Общее:


* REST API с версионированием — единый для всех клиентов (iOS, веб-приложение, лендинг)
* Язык/стек — на усмотрение команды
* БД: PostgreSQL — основное хранилище
* Хранилище файлов: S3-совместимое (российский провайдер)
* Очереди задач: Redis + воркеры


Авторизация и сессии:


Пользователи (iOS):


* SMS OTP через российского провайдера
* JWT-токены с refresh-логикой
* Sign in with Apple (обязательно для App Store), Google OAuth


Пользователи (веб-приложение):


* Email + пароль (bcrypt, мин. 8 символов)
* Google OAuth
* Сессия в httpOnly cookie (не localStorage)
* CSRF-токен для форм
* WebAuthn для повторного входа (если браузер поддерживает)


Сотрудники (AdminPanel):


* Логин + пароль
* Сессии с фиксированным TTL, принудительный выход при смене пароля
* Отдельная база учётных записей


Интеграция с VFS Global API:


* Получение списка доступных слотов
* Бронирование слота за пользователем
* Отмена бронирования слота
* Периодическая синхронизация данных о слотах (расписание — настраивается)


Генерация PDF анкеты VFS Global:


* Бэкенд формирует PDF по шаблону VFS Global с подставленными данными пользователя
* PDF доступен для скачивания, печати и отправки на email


Управление заявками, хранение файлов, OCR, платежи, уведомления, слоты, страховка, курс валют, аудит-лог:
Уведомления
* Push (iOS): APNs
* Push (веб): Web Push API (VAPID)
* Email: SMTP-сервис с шаблонами
Безопасность и соответствие
* Хранение данных на серверах в РФ (152-ФЗ)
* Шифрование в покое (AES-256) и в транзите (TLS 1.3)
* iOS: Face ID / Touch ID; веб: WebAuthn
* Автовыход из сессии после 15 мин неактивности
* Доступ к файлам только через временные подписанные ссылки
* CSRF-защита для веб-сессий
* Content Security Policy для лендинга и веб-приложения
