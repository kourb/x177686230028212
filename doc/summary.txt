Summary: ТЗ «Визовый Ассистент» v4.0 (26.03.2026)

1) Продукт и охват
- Платформы: iOS приложение, пользовательское веб-приложение, публичный SEO-лендинг, админ-панель.
- ЦА: граждане РФ для оформления шенгенских и других виз, включая групповые заявки; отдельный модуль страховки.
- Ключевой сценарий: от скана паспорта до готового пакета документов и (для шенгена) записи в визовый центр.

2) Этапы разработки
- Этап 1: iOS + админ-панель (основной рабочий контур).
- Этап 2: лендинг + веб-приложение + веб-специфичные backend endpoint-ы (email auth, WebAuthn, Web Push, browser file delivery).

3) Публичный лендинг (marketing site)
- Страницы: /, /insurance, /countries, /faq, /blog, /login, /register.
- Цель: конверсия в установки приложения и регистрации веб-версии.
- Требования: SSR/SSG, SEO (OpenGraph, Schema.org), Яндекс.Метрика, Core Web Vitals (LCP<2.5s, CLS<0.1, FID<100ms), адаптив.

4) Веб-приложение (порт iOS флоу)
- Desktop-first UX: левый sidebar, breadcrumbs, вертикальный stepper, drag&drop документов, preview/help колонка на широких экранах.
- Tablet/mobile адаптации: collapse sidebar, one-column flow, table->card, mobile tab bar.
- Web auth: email+password, Google OAuth, httpOnly cookie session, CSRF, WebAuthn (если поддерживается).
- Браузерные требования: Chrome/Firefox/Safari/Edge 110+ (Safari 16+), Lighthouse Performance>=80, Accessibility>=90.

5) Визовые типы и динамические анкеты
- С v4.0 мультивизовый режим: шенген + другие страны (добавляются через админку).
- Для шенгена: стандартные типы и фиксированные поля.
- Для остальных: полностью кастомные поля анкеты (text/date/dropdown/file/checkbox/number), настраиваемые в админке без релиза.
- Слоты визовых центров - только для шенгена.

6) VFS Global анкета
- В админке toggle «Анкета VFS Global».
- При включении: отдельный шаг после базовой анкеты (только шенген), автоподстановка данных, realtime validation, draft autosave.
- Выходы: предпросмотр, печать, PDF download, отправка PDF на email.

7) Пользовательский флоу (0-8)
- 0: авторизация (SMS OTP обязательно, Apple/Google доп.; в вебе также email/password + Google OAuth).
- 1: паспорт (OCR MRZ, автозаполнение, правка, несколько заявителей).
- 2: выбор типа визы + анкета (для не-шенгена поля из конструктора).
- 2a: VFS анкета (условно).
- 2b: страховка (загрузка своей или покупка внутри флоу).
- 3: фото (требования, ML-проверка, автообрезка, отдельно на каждую заявку/заявителя).
- 4: самопроверка + отправка менеджеру.
- 5: оплата сервисного сбора/страховки через ЮMoney.
- 6: ожидание проверки (push/email, возврат на шаг при правках).
- 7: генерация PDF + выбор центра/слота (только шенген).
- 8: финальные инструкции, карта, календарь (iOS EventKit, web .ics).

8) Отдельный модуль страховки
- Самостоятельный сценарий с главного экрана без визы.
- Параметры поездки -> данные застрахованных -> выбор программы (desktop: таблица+фильтры) -> оплата ЮMoney -> получение PDF полиса.
- Полис сохраняется в профиле и может автоприкрепляться к визовой анкете.

9) Админ-панель
- Роли: менеджер, старший менеджер, администратор.
- Модули: заявки, слоты/визовые центры (VFS API + ручной режим + автоприсвоение FIFO), пользователи, платежи, страховки, настройки.
- В настройках: типы виз, конструктор полей анкет, toggle VFS, PDF шаблоны, сотрудники, уведомления, валидации.

10) Backend и инфраструктура
- Единый versioned REST API для iOS/web/landing.
- DB: PostgreSQL; files: S3-compatible storage (РФ); queues: Redis + workers.
- Интеграции: SMS OTP, Apple/Google auth, APNs, Web Push (VAPID), email SMTP, ЮMoney, VFS Global API.
- Безопасность/комплаенс: 152-ФЗ (РФ), TLS 1.3, AES-256 at rest, signed URLs, CSRF, CSP, session idle timeout 15 мин.

11) Монетизация и KPI
- Монетизация: сервисный сбор, срочная обработка, комиссия со страховок.
- KPI: conversion >15%, анкета <12 мин, OCR >95%, photo pass >80% first try, CWV green zone, web Lighthouse>=80.
